## 3. LAYER-BY-LAYER BREAKDOWN

### LAYER 1: Constants (What files to read)
**File**: `src/cnnClassifier/constants/__init__.py`
```
CONFIG_FILE_PATH = Path("config/config.yaml")
PARAMS_FILE_PATH = Path("params.yaml")
```
- **Role**: Single source of truth for where config files live.
- **Used by**: `configuration.py`

---

### LAYER 2: Entity (Data shapes / contracts)
**File**: `src/cnnClassifier/entity/config_entity.py`

Defines **frozen dataclasses** — one per pipeline stage:

| Dataclass               | Stage | Fields                                                                           |
|--------------------------|-------|---------------------------------------------------------------------------------|
| `DataIngestionConfig`    | 1     | root_dir, source_URL, local_data_file, unzip_dir, split_dir, train/val/test_dir, split_ratio |
| `PrepareBaseModelConfig` | 2     | root_dir, base_model_path, updated_base_model_path, params_*                   |
| `TrainingConfig`         | 3     | root_dir, trained_model_path, updated_base_model_path, training_data, params_* , mlflow_uri |
| `EvaluationConfig`       | 4     | path_of_model, training_data, all_params, params_*, mlflow_uri                 |
| `CTGateConfig`           | 5     | root_dir, data_dir, model_path, params_*, mlflow_uri                           |

- **Role**: Type-safe configuration containers. Every component receives exactly the config it needs.
- **Used by**: `configuration.py` (creates them), `components/*.py` (consumes them)

---

### LAYER 3: Utils (Reusable helpers)
**File**: `src/cnnClassifier/utils/common.py`

| Function              | Purpose                                     |
|-----------------------|---------------------------------------------|
| `read_yaml(path)`     | Reads YAML → returns `ConfigBox` (dot-access dict) |
| `create_directories(list)` | Creates dirs with logging                 |
| `save_json(path, data)` | Saves dict to JSON                         |
| `load_json(path)`     | Loads JSON → `ConfigBox`                    |
| `get_size(path)`      | Returns file size in KB                     |
| `decodeImage(str, path)` | Decodes base64 image (used by Flask)      |
| `encodeImageIntoBase64(path)` | Encodes image to base64                |
| `save_bin` / `load_bin` | Joblib serialization                      |

- **Role**: Pure utility functions. No business logic.
- **Used by**: `configuration.py`, `components/*`, `app.py`

---

### LAYER 4: Configuration Manager (YAML → Dataclass)
**File**: `src/cnnClassifier/config/configuration.py`

```
class ConfigurationManager:
    def __init__(self):
        self.config = read_yaml(CONFIG_FILE_PATH)    # config/config.yaml
        self.params = read_yaml(PARAMS_FILE_PATH)     # params.yaml
        create_directories([self.config.artifacts_root])

    def get_data_ingestion_config()     → DataIngestionConfig
    def get_prepare_base_model_config() → PrepareBaseModelConfig
    def get_training_config()           → TrainingConfig
    def get_evaluation_config()         → EvaluationConfig
    def get_ct_gate_config()            → CTGateConfig
```

**Pattern**: One `get_*_config()` method per pipeline stage. Each method:
1. Reads the relevant section from `self.config` (config.yaml)
2. Reads hyperparameters from `self.params` (params.yaml)
3. Creates artifact directories
4. Returns a frozen dataclass

- **Role**: Central bridge between YAML configs and typed Python objects.
- **Used by**: Every `pipeline/stage_*.py` file

---

### LAYER 5: Components (Actual ML Logic)
**Directory**: `src/cnnClassifier/components/`

Each component receives its Config dataclass and does the actual work:

| Component File              | Class           | Config Received         | Methods                                       | Artifacts Produced                                    |
|-----------------------------|-----------------|-------------------------|-----------------------------------------------|------------------------------------------------------|
| `data_ingestion.py`         | `DataIngestion`  | `DataIngestionConfig`   | `download_file()`, `extract_zip_file()`, `split_data()` | `artifacts/data_ingestion/CT-scan-data/{train,val,test}/` |
| `prepare_base_model.py`     | `PrepareBaseModel` | `PrepareBaseModelConfig` | `get_base_model()`, `update_base_model()`     | `artifacts/prepare_base_model/base_model_updated.h5` |
| `model_trainer.py`          | `Training`       | `TrainingConfig`        | `get_base_model()`, `train_valid_generator()`, `train()` | `artifacts/training/model.h5`, `model/model.h5`     |
| `model_evaluation_mlflow.py` | `Evaluation`    | `EvaluationConfig`      | `evaluation()`, `calculate_advanced_metrics()`, `save_score()`, `log_into_mlflow()` | `scores.json`, MLflow logs          |
| `ct_gate_trainer.py`        | `CTGateTrainer`  | `CTGateConfig`          | `train()`                                     | `artifacts/ct_gate/ct_gate_model.h5`, `model/ct_gate_model.h5` |

- **Role**: ALL ML/DL code lives here. This is the only layer you need to heavily modify for a new project.
- **Used by**: `pipeline/stage_*.py`

---

### LAYER 6: Pipeline Stages (Thin Orchestrators)
**Directory**: `src/cnnClassifier/pipeline/`

Every stage file follows the **exact same pattern**:

```python
from cnnClassifier.config.configuration import ConfigurationManager
from cnnClassifier.components.{component} import {ComponentClass}
from cnnClassifier import logger

STAGE_NAME = "Stage Name"

class {StagePipeline}:
    def __init__(self):
        pass

    def main(self):
        config = ConfigurationManager()
        stage_config = config.get_{stage}_config()
        component = {ComponentClass}(config=stage_config)
        component.method_1()
        component.method_2()
        component.method_3()

if __name__ == '__main__':
    try:
        logger.info(f">>>>>> stage {STAGE_NAME} started <<<<<<")
        obj = {StagePipeline}()
        obj.main()
        logger.info(f">>>>>> stage {STAGE_NAME} completed <<<<<<")
    except Exception as e:
        logger.exception(e)
        raise e
```

**Each pipeline file is ~30 lines. The pattern never changes.** It:
1. Creates `ConfigurationManager()`
2. Gets the stage-specific config dataclass
3. Instantiates the component with that config
4. Calls the component's methods in order

---

### LAYER 7: Entry Points

| File       | Purpose                                                              |
|------------|----------------------------------------------------------------------|
| `main.py`  | **Training entry point** — imports all stage pipelines, runs them 1→2→3→4 sequentially |
| `app.py`   | **Serving entry point** — Flask app that uses `pipeline/prediction.py` for inference |

**main.py pattern**:
```python
for each stage:
    try:
        logger.info(f">>>>>> stage {STAGE_NAME} started <<<<<<")
        obj = StagePipeline()
        obj.main()
        logger.info(f">>>>>> stage {STAGE_NAME} completed <<<<<<")
    except Exception as e:
        logger.exception(e)
        raise e
```

**app.py pattern**:
```python
app = Flask(__name__)
clApp = ClientApp()  # loads PredictionPipeline once

@app.route("/predict", methods=['POST'])
def predictRoute():
    image = request.json['image']
    decodeImage(image, clApp.filename)
    result = clApp.classifier.predict()
    return jsonify(result)
```

## 5. DATA FLOW THROUGH THE PIPELINE

```
Stage 1: DATA INGESTION
────────────────────────
  INPUT:  Google Drive URL (from config.yaml)
  ACTION: download_file() → extract_zip_file() → split_data()
  OUTPUT: artifacts/data_ingestion/CT-scan-data/{train,val,test}/{class1,class2}/

          │
          ▼

Stage 2: PREPARE BASE MODEL
────────────────────────────
  INPUT:  Hyperparams (from params.yaml)
  ACTION: Download pretrained backbone → Add custom head → Freeze backbone → Save weights
  OUTPUT: artifacts/prepare_base_model/base_model_updated.h5

          │
          ▼

Stage 3: MODEL TRAINING
─────────────────────────
  INPUT:  base_model_updated.h5 + training data + hyperparams
  ACTION: Warmup training (head only) → Fine-tuning (unfreeze top layers)
          Logs to MLflow. Calculates class weights.
  OUTPUT: artifacts/training/model.h5  →  copied to  →  model/model.h5

          │
          ▼

Stage 4: MODEL EVALUATION
──────────────────────────
  INPUT:  model.h5 + test data
  ACTION: Predict on test set → Find optimal threshold → Calculate precision/recall/F1/AUC
          Logs to MLflow. Saves confusion matrix.
  OUTPUT: scores.json (all metrics including optimal_threshold)

          │
          ▼

Stage 5: CT GATE TRAINING (Optional / Domain-Specific)
───────────────────────────────────────────────────────
  INPUT:  CT vs non-CT training data
  ACTION: Train lightweight gate model
  OUTPUT: artifacts/ct_gate/ct_gate_model.h5  →  copied to  →  model/ct_gate_model.h5

          │
          ▼

PREDICTION (Serving)
────────────────────
  INPUT:  Base64-encoded image from user
  ACTION: Gate check (is it valid input?) → Main model prediction → Apply optimal threshold
  OUTPUT: JSON response {label, probability, confidence}
```

## 6. HOW TO REPRODUCE FOR A NEW PROJECT

### Step-by-step recipe:

#### A. Scaffold the project
Run `template.py` (or copy the tree). It creates all empty files:
```python
# template.py creates these paths:
.github/workflows/.gitkeep
src/{project_name}/__init__.py
src/{project_name}/components/__init__.py
src/{project_name}/utils/__init__.py
src/{project_name}/config/__init__.py
src/{project_name}/config/configuration.py
src/{project_name}/pipeline/__init__.py
src/{project_name}/entity/__init__.py
src/{project_name}/constants/__init__.py
config/config.yaml
dvc.yaml
params.yaml
requirements.txt
setup.py
research/trials.ipynb
templates/index.html
```

#### B. Define your stages (decide what pipeline you need)
Example stages for a different project:
```
Stage 1: Data Ingestion          (download + split)
Stage 2: Feature Engineering      (preprocessing)
Stage 3: Model Building           (architecture)
Stage 4: Model Training           (train loop)
Stage 5: Model Evaluation         (metrics + MLflow)
```

#### C. Fill in files in this exact order:

```
ORDER OF IMPLEMENTATION:
────────────────────────

1. config/config.yaml         ← Define ALL paths and URLs for your stages
2. params.yaml                ← Define ALL hyperparameters
3. entity/config_entity.py    ← Create one @dataclass per stage
4. constants/__init__.py      ← Just point to config.yaml and params.yaml (copy as-is)
5. utils/common.py            ← Copy as-is (read_yaml, create_directories, save_json, etc.)
6. __init__.py                ← Copy as-is (logger setup)
7. config/configuration.py    ← Add one get_*_config() method per stage
8. components/{stage}.py      ← WRITE YOUR ACTUAL ML CODE HERE (one class per stage)
9. pipeline/stage_*.py        ← Copy the pattern, swap class names (30 lines each)
10. pipeline/prediction.py    ← Write inference logic
11. main.py                   ← Import all pipelines, run sequentially
12. app.py                    ← Flask app using prediction.py
13. dvc.yaml                  ← Define stages with deps/outs/params
14. Dockerfile                ← Copy as-is
15. .github/workflows/main.yaml ← Copy, update secrets
16. setup.py                  ← Update project name
17. requirements.txt          ← List your dependencies
```

#### D. What to change vs what stays the same:

| File                      | Change?   | What to modify                                              |
|---------------------------|-----------|-------------------------------------------------------------|
| `template.py`             | Minimal   | Just change `project_name` variable                         |
| `constants/__init__.py`   | **No**    | Always the same 2 lines                                     |
| `utils/common.py`         | **No**    | Copy as-is, add custom utils only if needed                 |
| `__init__.py` (logger)    | **No**    | Copy as-is                                                  |
| `config/config.yaml`      | **YES**   | New paths, new URLs, new stage sections                     |
| `params.yaml`             | **YES**   | New hyperparameters for your model                          |
| `entity/config_entity.py` | **YES**   | New dataclasses matching your stages                        |
| `config/configuration.py` | **YES**   | New `get_*_config()` methods matching your stages            |
| `components/*.py`         | **YES**   | **This is where 90% of your new code goes**                 |
| `pipeline/stage_*.py`     | Minimal   | Same pattern, just swap imports and class names              |
| `pipeline/prediction.py`  | **YES**   | New inference logic for your model                           |
| `main.py`                 | Minimal   | Same pattern, add/remove stages                              |
| `app.py`                  | Minimal   | Same pattern, maybe change routes                            |
| `dvc.yaml`                | **YES**   | New stages, new deps/outs/params                             |
| `Dockerfile`              | **No**    | Copy as-is (or tweak base image)                             |
| `.github/workflows`       | Minimal   | Same structure, update secrets for your AWS account          |
| `setup.py`                | Minimal   | Change project name, author info                             |
| `requirements.txt`        | **YES**   | Your project's dependencies                                  |

## 7. DVC PIPELINE SPECIFICATION

```yaml
# dvc.yaml — Each stage defines:
#   cmd:    The Python script to run
#   deps:   Files that trigger re-run if changed
#   params: Hyperparams from params.yaml that trigger re-run
#   outs:   Output artifacts (DVC-cached)
#   metrics: Metric files (not cached, tracked by DVC for comparison)

stages:
  stage_name:
    cmd: python src/{package}/pipeline/stage_XX_{name}.py
    deps:
      - src/{package}/pipeline/stage_XX_{name}.py    # ← pipeline file
      - src/{package}/components/{component}.py       # ← component file
      - config/config.yaml                            # ← config
      # + any upstream artifacts this stage needs
    params:
      - PARAM_1
      - PARAM_2
    outs:
      - artifacts/{stage_name}/{output}
    metrics:
      - scores.json:
          cache: false    # (only on eval stage)
```

**Key rule**: deps include BOTH the pipeline file AND the component file. If you change the component logic, DVC knows to re-run.

**Stage dependency chain via artifacts**:
```
Stage 1 outs: artifacts/data_ingestion/data/
```

## UNIVERSAL PATTERN (The Golden Rule)

```
For EVERY pipeline stage, the flow is ALWAYS:

    config.yaml + params.yaml
           │
           ▼
    ConfigurationManager.get_{stage}_config()
           │
           ▼
    {Stage}Config  (frozen @dataclass)
           │
           ▼
    Component({Stage}Config)
           │
           ▼
    component.method_1()
    component.method_2()
    component.method_3()
           │
           ▼
    artifacts/{stage}/{output}
```

**This pattern is the same for 1 stage or 100 stages. To add a new stage:**
1. Add section in `config.yaml`
2. Add params in `params.yaml` (if needed)
3. Add `@dataclass` in `config_entity.py`
4. Add `get_*_config()` in `configuration.py`
5. Create `components/{new_stage}.py` with actual logic
6. Create `pipeline/stage_XX_{name}.py` (copy-paste pattern)
7. Add stage in `dvc.yaml`
8. Import and run in `main.py`

**That's it. The architecture scales linearly.**
